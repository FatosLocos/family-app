Lokale probe installeren

  1. Kies een apparaat dat thuis altijd aanstaat.Een Raspberry Pi is het meest geschikt; een Mac of laptop werkt ook zolang deze aan en verbonden blijft met het thuisnetwerk.
  2. Download de probe hierboven.Pak family-app-local-probe.zip uit op het gekozen apparaat.
  3. Installeer de afhankelijkheden.python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
  4. Maak een pairing-code.Kies Nieuwe probe koppelen. De code is tien minuten geldig en slechts eenmaal te gebruiken.
  5. Koppel de probe..venv/bin/python -m family_app_probe.main pair --server {{ probe_server_url }} --code FAP-... --name "Woonkamer probe"
  6. Koppel Hue lokaal, indien aanwezig.Druk eerst op de knop van de Hue Bridge:.venv/bin/python -m family_app_probe.main hue-link --bridge https://192.168.1.20
  7. Controleer de verbinding..venv/bin/python -m family_app_probe.main run --onceStart daarna de continue verbinding:.venv/bin/python -m family_app_probe.main run
  8. Sta lokale Sonos-notificaties toe.Wanneer je firewall daarom vraagt, geef de probe/Python toegang op het lokale netwerk. Sonos stuurt volume-, afspeel- en groepswijzigingen dan direct naar de probe.
  9. Controleer deze pagina.De probe wordt als Verbonden getoond. Bij Sonos staat live-updates met het aantal actieve abonnementen. Hue en Sonos synchroniseren lokaal; andere SSDP- en mDNS-vondsten verschijnen uitsluitend als suggesties.
  10. Toegang stoppen.Gebruik het ontkoppel-icoon naast de probe. Het token wordt dan onmiddellijk ongeldig.

De probe opent zelf een beveiligde verbinding naar de app. Er zijn geen inkomende poorten op de router nodig.

Philips Hue koppelen

  1. Open het Hue Developer Portal, kies rechtsboven Remote Hue API appids en daarna Add new Remote Hue API app.
  2. Registreer als redirect-URL: {{ hue_redirect_url }}.
  3. Vul de client ID, client secret en eventueel application ID van die app in en kies Opslaan.
  4. Kies Koppel Philips Hue, rond de Hue-aanmelding af, druk op de fysieke Bridge-knop en kies Voltooien.

Sonos koppelen

  1. Open het Sonos Developer Portal en maak een Control integration met credentials aan.
  2. Registreer deze OAuth redirect-URL: {{ sonos_redirect_url }}.
  3. Registreer ook deze event callback URL: {{ sonos_event_callback_url }}.
  4. Neem de integration key en secret over, schakel Sonos-events ontvangen in, sla op en kies Koppel Sonos.
  5. Geef de scope playback-control-all toestemming. De app haalt daarna groepen op en vernieuwt eventsubscriptions automatisch.

De event callback is alleen voor de publieke HTTPS-app. Hij controleert elke Sonos-signatuur voordat een apparaatstatus wordt bijgewerkt.

Google Home koppelen

  1. Meld je aan in de Google Device Access Console en maak een Device Access-project.
  2. Maak in Google Cloud Credentials een OAuth-webclient en schakel de Smart Device Management API in.
  3. Registreer als redirect-URL: {{ google_home_redirect_url }}.
  4. Koppel de OAuth client aan het Device Access-project, vul client ID, secret en project ID in en kies Koppel Google Home.
  5. Selecteer in Google de woning en ondersteunde Nest-apparaten.

Dit gebruikt de officiële Device Access API: Google Nest-apparaten en -traits, niet een onbeperkte API voor alle apparaten die toevallig in de Google Home-app staan.

LG ThinQ koppelen

  1. Open LG Developer en ga daar door naar Smart Solution API Developer om een persoonlijke ThinQ API-app voor jouw regio te maken.
  2. Registreer als redirect-URL: {{ lg_thinq_redirect_url }}.
  3. Neem client ID, secret, authorization URL, token URL, API base URL en apparatenpad uit de appdocumentatie over.
  4. Sla op en kies Koppel LG ThinQ.

LG levert endpoints per app/regio. Daarom worden ze expliciet ingesteld in plaats van een mogelijk onjuist, niet-officieel endpoint te gebruiken.
