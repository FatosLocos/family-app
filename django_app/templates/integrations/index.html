Lokale probe installeren

  1. Kies een apparaat dat thuis altijd aanstaat.Een Raspberry Pi is het meest geschikt; een Mac of laptop werkt ook zolang deze aan en verbonden blijft met het thuisnetwerk.
  2. Download de probe hierboven.Pak family-app-local-probe.zip uit op het gekozen apparaat.
  3. Installeer de afhankelijkheden.Gebruik Python 3.12 of 3.13. Bluetooth LE-scans werken voorlopig niet met Python 3.14.python3.12 -m venv .venv && .venv/bin/pip install -r requirements.txt
  4. Maak een pairing-code.Kies Nieuwe probe koppelen. De code is tien minuten geldig en slechts eenmaal te gebruiken.
  5. Koppel de probe..venv/bin/python -m family_app_probe.main pair --server {{ probe_server_url }} --code FAP-... --name "Woonkamer probe"
  6. Koppel Hue lokaal, indien aanwezig.Druk eerst op de knop van de Hue Bridge{% if local_hue_bridge %}. De lokale probe vond hem op {{ local_hue_bridge.address }}{% endif %}. Kies daarna op deze instellingenpagina Bridge lokaal koppelen. Zonder ontdekte bridge kan dit ook handmatig:.venv/bin/python -m family_app_probe.main hue-link --bridge https://{{ local_hue_bridge.address|default:'192.168.1.20' }}
  7. Koppel Philips TV lokaal, indien aanwezig.Gebruik het lokale IP-adres van de TV. De probe vraagt om de pincode die de TV tijdens pairing toont; de inloggegevens blijven alleen op de probe..venv/bin/python -m family_app_probe.main philips-tv-link --host 192.168.1.234
  8. Koppel Nest Protect lokaal (experimenteel).De officiële Google API ondersteunt geen Nest Protect. Open de Nest Protect-instructie in de probe-README, haal de volledige Google iframerpc-Request URL en tijdelijke cookies op en plak die alleen op de probe:.venv/bin/python -m family_app_probe.main nest-protect-link --issue-token "https://accounts.google.com/..."Cookies worden verborgen ingevoerd, blijven op de probe en moeten na een Google-uitlog of wachtwoordwijziging opnieuw worden ingesteld.
  9. Controleer de verbinding..venv/bin/python -m family_app_probe.main run --onceDit stuurt alleen de inventaris en stopt daarna. Start vervolgens de continue verbinding:.venv/bin/python -m family_app_probe.main run
  10. Sta lokale Sonos-notificaties toe.Wanneer je firewall daarom vraagt, geef de probe/Python toegang op het lokale netwerk. Sonos stuurt volume-, afspeel- en groepswijzigingen dan direct naar de probe.
  11. Controleer deze pagina.De probe wordt als Verbonden getoond. Hue en Sonos synchroniseren lokaal; Nest Protect is alleen-lezen en meldt rook-, CO-, hitte- en voedingsstatus.
  12. Toegang stoppen.Gebruik het ontkoppel-icoon naast de probe. Het token wordt dan onmiddellijk ongeldig.

De probe opent zelf een beveiligde verbinding naar de app. Er zijn geen inkomende poorten op de router nodig.

OpenClaw koppelen (MCP)

OpenClaw praat met FamilyApp via een kleine "MCP-bridge" die op dezelfde server draait als FamilyApp zelf en de bestaande, beveiligde OpenClaw-API aanroept. Eén bridge kan meerdere gezinsleden bedienen: elk gezinslid met een eigen OpenClaw-installatie maakt zijn eigen token en registreert dat met zijn eigen header. De bridge-installatie (stap 2–4) is maar één keer nodig voor de hele VPS.

  1. Maak je eigen token.Vink hierboven aan welke rechten dit token krijgt (bijvoorbeeld alleen boodschappen lezen, of ook taken aanmaken) en kies Token aanmaken. Het token wordt één keer getoond — bewaar het direct. Ieder gezinslid met een eigen OpenClaw-koppeling doet dit voor zichzelf, met de rechten die bij hen passen.
  2. Installeer de MCP-bridge (eenmalig per VPS).De broncode staat al in de FamilyApp-repo onder mcp_bridge/.cd /opt/family-app && sudo python3 -m venv .venv-mcp-bridge && sudo .venv-mcp-bridge/bin/pip install -r mcp_bridge/requirements.txt
  3. Maak een systemd-service (eenmalig per VPS).Zet dit in /etc/systemd/system/family-app-mcp-bridge.service — zonder token, want dat komt straks per gezinslid mee via OpenClaw's eigen registratie:[Unit]
    Description=FamilyApp MCP bridge
    After=network.target

    [Service]
    User=deploy
    Environment=FAMILY_APP_BASE_URL=http://127.0.0.1:8088
    ExecStart=/opt/family-app/.venv-mcp-bridge/bin/python /opt/family-app/mcp_bridge/server.py
    Restart=on-failure

    [Install]
    WantedBy=multi-user.target
  4. Start de service (eenmalig per VPS).sudo systemctl daemon-reload && sudo systemctl enable --now family-app-mcp-bridgeControleer met sudo systemctl status family-app-mcp-bridge — moet active (running) tonen.
  5. Registreer de MCP-server bij jouw OpenClaw, met je eigen token.openclaw mcp add family-app-taken --url http://127.0.0.1:8899/mcp --transport streamable-http --header "Authorization=Bearer <jouw token uit stap 1>"
    openclaw mcp probe family-app-taken
    openclaw mcp reload    De probe-uitvoer moet de tools vandaag, taak_toevoegen, taak_afronden, boodschappen, boodschap_toevoegen, huis, huis_bedienen, agenda, afspraak_toevoegen, geld, meldingen, meldingen_afgeleverd , dropbox_overzicht, dropbox_map en dropbox_zoeken tonen.
  6. Testen.Stuur OpenClaw een bericht zoals "wat staat er deze week op de agenda", "voeg melk toe aan de boodschappen", "doe het licht in de woonkamer uit", "zet donderdag 19:00-20:00 tandarts in de agenda" of "hoeveel geef ik deze maand nog uit aan boodschappen". In de activiteitenlijst hierboven verschijnt jouw naam bij de actie.
  7. Optioneel: proactieve meldingen inschakelen.Zet hierboven aan welke meldingscategorieën je via WhatsApp wilt ontvangen (vereist een token met de scopes "Openstaande meldingen lezen" en "Meldingen als afgeleverd markeren"). Registreer daarna een periodieke cron-job in OpenClaw die zelf checkt en aflevert:openclaw cron add --every 30m --to <jouw eigen nummer, bijv. +31612345678> --message "Roep de meldingen-tool van FamilyApp aan. Als er nieuwe meldingen zijn, vat ze kort en vriendelijk samen in het Nederlands en lever dat af. Roep daarna meldingen_afgeleverd aan met de ids die je hebt afgeleverd. Als er niets nieuws is, doe helemaal niets." --tools meldingen,meldingen_afgeleverd --best-effort-deliverJe eigen nummer vind je met openclaw directory self --channel whatsapp.
  8. Bij een nieuw of ingetrokken token.openclaw mcp configure ondersteunt geen header-wijzigingen — verwijder en registreer opnieuw met het nieuwe token:openclaw mcp unset family-app-taken
    openclaw mcp add family-app-taken --url http://127.0.0.1:8899/mcp --transport streamable-http --header "Authorization=Bearer <nieuw token>"
    openclaw mcp probe family-app-taken
    openclaw mcp reload

De MCP-bridge is alleen bereikbaar op 127.0.0.1 van de VPS zelf — nooit publiek toegankelijk.

Philips Hue koppelen

  1. Open het Hue Developer Portal, kies rechtsboven Remote Hue API appids en daarna Add new Remote Hue API app.
  2. Registreer als redirect-URL: {{ hue_redirect_url }}.
  3. Vul de client ID, client secret en eventueel application ID van die app in en kies Opslaan.
  4. Kies Koppel Philips Hue, rond de Hue-aanmelding af, druk op de fysieke Bridge-knop en kies Voltooien.

Sonos koppelen

  1. Open het Sonos Developer Portal en maak een Control integration met credentials aan.
  2. Registreer deze OAuth redirect-URL: {{ sonos_redirect_url }}.
  3. Registreer ook deze event callback URL: {{ sonos_event_callback_url }}.
  4. Neem de integration key en secret over, schakel Sonos-events ontvangen in, sla op en kies Koppel Sonos.
  5. Geef de scope playback-control-all toestemming. De app haalt daarna groepen op en vernieuwt eventsubscriptions automatisch.

De event callback is alleen voor de publieke HTTPS-app. Hij controleert elke Sonos-signatuur voordat een apparaatstatus wordt bijgewerkt.

Spotify Connect koppelen

  1. Open het Spotify Developer Dashboard en maak een app aan.
  2. Open de appinstellingen en registreer als redirect-URL: {{ spotify_redirect_url }}.
  3. Neem de client ID en client secret over, kies Opslaan en daarna Koppel Spotify.
  4. Geef toestemming voor de afspeelrechten. Daarna verschijnen de apparaten die Spotify Connect zelf beschikbaar stelt onder Huis.

Voor afspelen, pauzeren, overslaan, volume en overdracht is Spotify Premium nodig. Spotify Connect toont alleen doelen die op dat moment beschikbaar zijn.

Dropbox koppelen

  1. Open de Dropbox App Console en kies Create app.
  2. Kies Scoped access en als toegangstype App folder — FamilyApp krijgt dan alleen toegang tot zijn eigen map, niet je hele Dropbox.
  3. Ga naar het tabblad Permissions en vink files.metadata.read en account_info.read aan. Kies daarna Submit.
  4. Ga naar het tabblad Settings en registreer als redirect-URL: {{ dropbox_redirect_url }}.
  5. Neem de App key en App secret over, kies Opslaan en daarna Koppel Dropbox.

FamilyApp leest alleen bestandsnamen, wijzigingsdatum en grootte — via een overzicht, mappen bekijken of zoeken op naam — nooit de inhoud, en er is geen mogelijkheid om bestanden te wijzigen of te verwijderen.

Home Connect koppelen

  1. Open het Home Connect Developer Portal en maak een OAuth-app.
  2. Registreer als redirect-URL: {{ home_connect_redirect_url }}.
  3. Neem de client ID en client secret over, kies Opslaan en daarna Koppel Home Connect.
  4. Meld je aan met hetzelfde Home Connect-account als waarin de Siemens-vaatwasser staat.

De app toont alleen wat jouw apparaat via de officiële API beschikbaar stelt. Programma starten of stoppen wordt pas als expliciete, bevestigde bediening toegevoegd.

Google Home koppelen

  1. Meld je aan in de Google Device Access Console en maak een Device Access-project.
  2. Maak in Google Cloud Credentials een OAuth-webclient en schakel de Smart Device Management API in.
  3. Registreer als redirect-URL: {{ google_home_redirect_url }}.
  4. Koppel de OAuth client aan het Device Access-project, vul client ID, secret en project ID in en kies Koppel Google Home.
  5. Voor live updates: maak in Google Cloud een Pub/Sub-topic en pull-subscription, geef sdm-publisher@googlegroups.com Publisher op het topic en maak een serviceaccount met alleen Pub/Sub Subscriber voor de subscription. Vul daarna de volledige subscriptionnaam en het serviceaccount JSON hierboven in.

Dit gebruikt de officiële Device Access API. Nest Protect-rook- en CO-melders worden door deze API niet ondersteund.

Smartcar koppelen

  1. Maak een app in het Smartcar Dashboard en registreer als redirect-URL: {{ smartcar_redirect_url }}.
  2. Open Configuration → Vehicle Access, selecteer de signalen en opdrachten die je wilt gebruiken en kies Publish. De app neemt deze selectie exact over.
  3. Neem het Connect client ID uit de appconfiguratie over. Gebruik hier niet een ID dat met client_ begint.
  4. Open daarna API Credentials, maak een API secret aan en vul de API client ID en secret hierboven in.
  5. Kies Opslaan en daarna Koppel Smartcar; meld je aan bij het automerk en kies de voertuigen opnieuw na elke wijziging aan Vehicle Access.

De API-secret blijft versleuteld op de server. Vergrendelen en ontgrendelen worden alleen getoond wanneer ze in Vehicle Access zijn geselecteerd en expliciet zijn toegestaan.

LG ThinQ koppelen

  1. Open LG Developer en ga daar door naar Smart Solution API Developer om een persoonlijke ThinQ API-app voor jouw regio te maken.
  2. Registreer als redirect-URL: {{ lg_thinq_redirect_url }}.
  3. Neem client ID, secret, authorization URL, token URL, API base URL en apparatenpad uit de appdocumentatie over.
  4. Sla op en kies Koppel LG ThinQ.

LG levert endpoints per app/regio. Daarom worden ze expliciet ingesteld in plaats van een mogelijk onjuist, niet-officieel endpoint te gebruiken.
